# 通用模型接入文档

更新时间：2026-07-19

本文档用于客户通过本平台统一接入图片生成、图片编辑和视频生成模型。绝大多数新模型上线后，客户只需要把请求里的 `model` 改成新的模型 ID，即可继续沿用本文档的接口和参数。

## 1. 基础信息

| 项目 | 内容 |
| --- | --- |
| Base URL | `http://154.40.44.244:3000` |
| 鉴权方式 | `Authorization: Bearer <YOUR_API_KEY>` |
| 视频生成 | `POST /v1/videos` |
| 视频任务查询 | `GET /v1/videos/{task_id}` |
| 视频兜底下载 | `GET /v1/videos/{task_id}/content` |
| 图片生成 | `POST /v1/images/generations` |
| 图片编辑 | `POST /v1/images/edits` |
| Gemini 参考图兼容 | `POST /v1/chat/completions` |

请求头：

```http
Authorization: Bearer <YOUR_API_KEY>
Content-Type: application/json
```

请妥善保管 API Key，不要放在前端代码、公开仓库或客户端 App 明文包中。

## 2. 接入原则

推荐客户侧按以下方式接入：

1. 基础文生视频、图生视频、文生图：只切换 `model`。
2. 参考图片、参考视频、参考音频、首尾帧：使用本文档的统一字段。
3. 新模型上线后，先看“模型能力矩阵”。如果新模型支持同类能力，通常只改 `model` 即可。
4. 不要向不支持的模型传对应素材，例如 `xq93-2.0` 不支持参考视频。
5. 视频任务是异步接口。提交成功只代表任务已接收，最终结果以查询接口为准。

## 3. 通用视频接口

### 3.1 创建视频任务

```http
POST /v1/videos
```

通用请求示例：

```bash
curl -X POST "http://154.40.44.244:3000/v1/videos" \
  -H "Authorization: Bearer <YOUR_API_KEY>" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "sd-720-pro",
    "prompt": "电影感写实画面，一位女孩在海边奔跑，自然光影，镜头缓慢推进，无字幕无水印",
    "seconds": "10",
    "duration": "10",
    "aspect_ratio": "16:9",
    "ratio": "16:9",
    "resolution": "720p",
    "reference_image_urls": [
      "https://example.com/reference.jpg"
    ],
    "reference_videos": [],
    "reference_audios": [],
    "first_frame_url": "",
    "last_frame_url": ""
  }'
```

最小文生视频请求：

```json
{
  "model": "sd-720-pro",
  "prompt": "电影感城市夜景，霓虹灯，镜头缓慢推进，无文字水印",
  "seconds": "10",
  "aspect_ratio": "16:9",
  "resolution": "720p"
}
```

### 3.2 视频请求字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `model` | string | 是 | 模型 ID。切换模型时主要修改此字段 |
| `prompt` | string | 是 | 视频提示词 |
| `seconds` | string/number | 否 | 视频秒数，推荐传字符串，如 `"10"` |
| `duration` | string/number | 否 | 与 `seconds` 兼容，建议和 `seconds` 保持一致 |
| `aspect_ratio` | string | 否 | 画面比例，如 `16:9`、`9:16`、`1:1`、`4:3` |
| `ratio` | string | 否 | `aspect_ratio` 的兼容字段 |
| `aspectRatio` | string | 否 | `aspect_ratio` 的兼容字段 |
| `size` | string | 否 | 输出尺寸，如 `1280x720`、`720x1280`、`1920x1080` |
| `resolution` | string | 否 | 分辨率，如 `720p`、`1080p`、`480p` |
| `reference_image_urls` | string[] | 否 | 参考图片 URL 数组 |
| `image_urls` / `images` | string[] | 否 | 参考图片兼容字段 |
| `image_url` / `image` | string | 否 | 单张参考图片兼容字段 |
| `input_reference` | string/string[] | 否 | 参考图兼容字段 |
| `reference_videos` | string[] | 否 | 参考视频 URL 数组 |
| `video_urls` / `videos` | string[] | 否 | 参考视频兼容字段 |
| `video_url` / `video` | string | 否 | 单个参考视频兼容字段 |
| `reference_audios` | string[] | 否 | 参考音频 URL 数组 |
| `audio_urls` / `audios` | string[] | 否 | 参考音频兼容字段 |
| `audio_url` / `audio` | string | 否 | 单个参考音频兼容字段 |
| `first_frame_url` | string | 否 | 首帧图片 URL |
| `first_frame` | string | 否 | 首帧兼容字段 |
| `last_frame_url` | string | 否 | 尾帧图片 URL |
| `last_frame` | string | 否 | 尾帧兼容字段 |
| `assets` | array | 否 | 高级素材数组，见下文 |
| `files` | array | 否 | 高级素材数组，部分模型支持 |
| `metadata` | object | 否 | 扩展参数 |
| `extra` / `extra_body` | object | 否 | 扩展参数 |

`canvas-seedance-fast-2.0-9img` 和 `canvas-seedance2-stable-channel2` 也走本通用视频接口，客户侧只需要把 `model` 改成对应模型 ID，其他字段继续沿用 `prompt`、`seconds`、`aspect_ratio`、`resolution` 和各类参考素材字段。

### 3.3 assets / files 素材数组

部分模型支持 `assets` 或 `files` 数组。建议新接入统一使用 URL 字段，只有在模型特殊要求时再使用数组。

普通参考素材：

```json
{
  "model": "sd-720-pro",
  "prompt": "参考素材生成电影感视频",
  "duration": "10",
  "assets": [
    { "type": "image", "url": "https://example.com/ref.png", "role": "reference" },
    { "type": "video", "url": "https://example.com/ref.mp4", "role": "reference" },
    { "type": "audio", "url": "https://example.com/ref.mp3", "role": "audio" }
  ]
}
```

首尾帧：

```json
{
  "model": "sd-720-fast",
  "prompt": "从首帧自然过渡到尾帧，动作流畅",
  "duration": "10",
  "referenceMode": "first_last_frame",
  "assets": [
    { "type": "image", "url": "https://example.com/first.png", "role": "first_frame" },
    { "type": "image", "url": "https://example.com/last.png", "role": "last_frame" }
  ]
}
```

### 3.4 视频任务提交返回

```json
{
  "id": "task_xxxxxxxxxxxxx",
  "task_id": "task_xxxxxxxxxxxxx",
  "object": "video",
  "model": "sd-720-pro",
  "status": "queued",
  "progress": 0,
  "created_at": 1784438896
}
```

建议按以下顺序读取任务 ID：

```text
id -> task_id -> taskId -> data.id -> data.task_id -> data.taskId
```

### 3.5 查询视频任务

```http
GET /v1/videos/{task_id}
```

请求示例：

```bash
curl -X GET "http://154.40.44.244:3000/v1/videos/<TASK_ID>" \
  -H "Authorization: Bearer <YOUR_API_KEY>"
```

成功返回示例：

```json
{
  "id": "task_xxxxxxxxxxxxx",
  "task_id": "task_xxxxxxxxxxxxx",
  "object": "video",
  "model": "sd-720-pro",
  "status": "completed",
  "progress": 100,
  "url": "https://example.com/result.mp4",
  "video_url": "https://example.com/result.mp4",
  "result_url": "https://example.com/result.mp4",
  "outputs": [
    {
      "url": "https://example.com/result.mp4",
      "download_url": "https://example.com/result.mp4",
      "type": "video"
    }
  ]
}
```

建议按以下顺序读取结果 URL：

```text
url -> video_url -> videoUrl -> result_url -> downloadUrl -> download_url -> outputVideoUrl -> output_url -> outputs[0].url -> outputs[0].download_url -> generated[0] -> data.url -> data.video_url -> data.result_url
```

常见状态：

| 状态 | 含义 |
| --- | --- |
| `queued`, `pending`, `submitted`, `waiting` | 排队中 |
| `processing`, `running`, `in_progress`, `generating` | 生成中 |
| `completed`, `succeeded`, `success`, `finished`, `done` | 已完成 |
| `failed`, `failure`, `error`, `cancelled`, `canceled` | 失败或取消 |

### 3.6 轮询建议

提交任务后建议等待 5 秒再开始查询。

推荐策略：

1. 前 1 分钟每 5 秒查询一次。
2. 1 分钟后每 10 秒查询一次。
3. 任务完成或失败后停止轮询。
4. 建议最长等待 15-30 分钟，长视频或高峰期可适当延长。
5. 不要重复提交同一个请求，否则会产生重复计费。

## 4. 通用图片接口

### 4.1 图片生成

```http
POST /v1/images/generations
```

请求示例：

```bash
curl -X POST "http://154.40.44.244:3000/v1/images/generations" \
  -H "Authorization: Bearer <YOUR_API_KEY>" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-image-2",
    "prompt": "高质量产品海报，白色背景，摄影棚布光，主体清晰，无文字",
    "size": "1024x1024",
    "n": 1,
    "response_format": "url"
  }'
```

### 4.2 图片参考 / 图片编辑

`gpt-image-2` 或支持图片编辑的模型可使用：

```http
POST /v1/images/edits
```

JSON 参考图示例：

```json
{
  "model": "gpt-image-2",
  "prompt": "参考图主体保持一致，改成赛博朋克海报风格",
  "image": [
    "https://example.com/reference.png"
  ],
  "size": "1024x1024",
  "response_format": "url"
}
```

也可以使用：

```json
{
  "model": "gpt-image-2",
  "prompt": "参考图主体保持一致，商业海报风格",
  "reference_images": [
    "https://example.com/reference.png"
  ],
  "size": "1024x1024"
}
```

### 4.3 Gemini 参考图

Gemini 图片模型做参考图时，建议使用 `POST /v1/chat/completions` 的多模态格式：

```bash
curl -X POST "http://154.40.44.244:3000/v1/chat/completions" \
  -H "Authorization: Bearer <YOUR_API_KEY>" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gemini-3.1-flash-image-preview",
    "messages": [
      {
        "role": "user",
        "content": [
          { "type": "text", "text": "根据参考图生成清爽商业海报，无文字" },
          {
            "type": "image_url",
            "image_url": {
              "url": "https://example.com/reference.png",
              "detail": "high"
            }
          }
        ]
      }
    ],
    "stream": false,
    "extra_body": {
      "aspect_ratio": "1:1",
      "resolution": "2K"
    }
  }'
```

### 4.4 图片请求字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `model` | string | 是 | 图片模型 ID |
| `prompt` | string | 是 | 图片描述或编辑要求 |
| `size` | string | 否 | 如 `1024x1024`、`1792x1024`、`1024x1792`、`2048x2048` |
| `n` | number | 否 | 生成数量，默认 `1` |
| `response_format` | string | 否 | `url` 或 `b64_json`，推荐 `url` |
| `quality` | string | 否 | 如 `low`、`medium`、`high`，Gemini 可用 `1K`、`2K`、`4K` |
| `background` | string | 否 | `opaque` 或 `transparent`，视模型支持情况而定 |
| `image` | string/string[] | 否 | 参考图 URL/base64/data URL |
| `reference_images` | string[] | 否 | 参考图 URL 数组 |
| `input_reference` | string/string[] | 否 | 参考图兼容字段 |
| `messages` | array | 否 | `/v1/chat/completions` 多模态请求使用 |
| `extra_body` | object | 否 | 额外参数，如 `aspect_ratio`、`resolution` |

## 6. 客户只切换模型的示例

客户可以保持请求结构不变，只替换 `model`：

```json
{
  "model": "seedance_2_fast",
  "prompt": "电影感写实画面，人物在雨夜街头缓慢回头，无字幕无水印",
  "duration": "10",
  "aspect_ratio": "16:9",
  "resolution": "720p",
  "reference_image_urls": [
    "https://example.com/person.jpg"
  ]
}
```

切换到另一个支持参考图的视频模型：

```json
{
  "model": "sd-720-pro",
  "prompt": "电影感写实画面，人物在雨夜街头缓慢回头，无字幕无水印",
  "duration": "10",
  "aspect_ratio": "16:9",
  "resolution": "720p",
  "reference_image_urls": [
    "https://example.com/person.jpg"
  ]
}
```

切换到不支持参考视频的模型时，应删除 `reference_videos`、`video_urls`、`videos` 等字段。

`viraldance431` 按次计费 3.5 元/次，支持文生视频、图生视频、首尾帧以及图片/视频/音频多模态参考。使用通用字段提交时，平台会将 `reference_image_urls`、`image_urls`、`images` 等图片字段转换为上游 `image_urls`，将 `reference_videos`、`video_urls`、`videos` 转换为 `video_reference`，将 `reference_audios`、`audio_urls`、`audios` 转换为 `audio_url` / `audio_reference`。该模型上游要求参考图最多 4 张，参考视频最多 3 个且总时长不超过 15 秒，参考音频最多 1 条且不超过 15 秒；任务完成后会返回可公网访问的 MP4 结果 URL。

## 7. JavaScript 通用视频示例

```js
const BASE_URL = "http://154.40.44.244:3000";
const API_KEY = process.env.API_KEY;

function getTaskId(body) {
  return body.id || body.task_id || body.taskId || body.data?.id || body.data?.task_id || body.data?.taskId;
}

function getVideoUrl(body) {
  return (
    body.url ||
    body.video_url ||
    body.videoUrl ||
    body.result_url ||
    body.downloadUrl ||
    body.download_url ||
    body.outputVideoUrl ||
    body.output_url ||
    body.outputs?.[0]?.url ||
    body.outputs?.[0]?.download_url ||
    body.generated?.[0] ||
    body.data?.url ||
    body.data?.video_url ||
    body.data?.result_url ||
    ""
  );
}

async function createVideo(payload) {
  const response = await fetch(`${BASE_URL}/v1/videos`, {
    method: "POST",
    headers: {
      Authorization: `Bearer ${API_KEY}`,
      "Content-Type": "application/json"
    },
    body: JSON.stringify(payload)
  });

  const body = await response.json();
  if (!response.ok) {
    throw new Error(body.error?.message || body.message || JSON.stringify(body));
  }
  return getTaskId(body);
}

async function pollVideo(taskId) {
  while (true) {
    const response = await fetch(`${BASE_URL}/v1/videos/${encodeURIComponent(taskId)}`, {
      headers: { Authorization: `Bearer ${API_KEY}` }
    });
    const body = await response.json();
    const status = String(body.status || body.data?.status || "").toLowerCase();
    const url = getVideoUrl(body);

    if (url) return url;
    if (["failed", "failure", "error", "cancelled", "canceled"].includes(status)) {
      throw new Error(body.failure_reason || body.error?.message || body.message || "video generation failed");
    }

    await new Promise((resolve) => setTimeout(resolve, 8000));
  }
}

async function main() {
  const taskId = await createVideo({
    model: "sd-720-pro",
    prompt: "电影感写实画面，女孩在海边奔跑，自然光影，无字幕无水印",
    duration: "10",
    aspect_ratio: "16:9",
    resolution: "720p"
  });
  const url = await pollVideo(taskId);
  console.log(url);
}

main().catch(console.error);
```

## 8. Python 通用视频示例

```python
import time
import requests

BASE_URL = "http://154.40.44.244:3000"
API_KEY = "YOUR_API_KEY"

headers = {
    "Authorization": f"Bearer {API_KEY}",
    "Content-Type": "application/json",
}

payload = {
    "model": "sd-720-pro",
    "prompt": "电影级写实画面，清晨山谷云雾流动，镜头缓慢推进，无字幕无水印",
    "duration": "10",
    "aspect_ratio": "16:9",
    "resolution": "720p",
}

submit = requests.post(f"{BASE_URL}/v1/videos", headers=headers, json=payload, timeout=300)
body = submit.json()
if submit.status_code >= 400:
    raise RuntimeError(body.get("error", {}).get("message") or body.get("message") or str(body))

task_id = body.get("id") or body.get("task_id") or body.get("taskId") or body.get("data", {}).get("id")

def pick_url(data):
    for key in ("url", "video_url", "videoUrl", "result_url", "downloadUrl", "download_url", "outputVideoUrl", "output_url"):
        if data.get(key):
            return data[key]
    outputs = data.get("outputs") or []
    if outputs and isinstance(outputs[0], dict):
        return outputs[0].get("url") or outputs[0].get("download_url") or ""
    generated = data.get("generated") or []
    if generated:
        return generated[0]
    nested = data.get("data") or {}
    for key in ("url", "video_url", "videoUrl", "result_url"):
        if nested.get(key):
            return nested[key]
    return ""

while True:
    poll = requests.get(f"{BASE_URL}/v1/videos/{task_id}", headers=headers, timeout=60)
    data = poll.json()
    url = pick_url(data)
    status = str(data.get("status") or (data.get("data") or {}).get("status") or "").lower()

    if url:
        print(url)
        break
    if status in ("failed", "failure", "error", "cancelled", "canceled"):
        raise RuntimeError(data.get("failure_reason") or data.get("error", {}).get("message") or "video generation failed")

    time.sleep(8)
```

## 9. 常见错误

| HTTP 状态码 | 说明 |
| --- | --- |
| 400 | 参数错误，例如缺少 `model`、缺少 `prompt`、时长超出模型范围、传入不支持的参考素材 |
| 401 | API Key 缺失或无效 |
| 403 | 余额不足、模型无权限、分组无权限或额度不足 |
| 404 | 任务不存在，或任务 ID 错误 |
| 409 | 任务尚未完成，无法下载内容 |
| 429 | 请求过于频繁 |
| 500/502 | 平台或模型服务异常 |

错误响应通常为：

```json
{
  "error": {
    "message": "错误说明",
    "type": "new_api_error",
    "code": "error_code"
  }
}
```

## 10. 新模型上线后的客户接入方式

当平台新增模型后，如果新模型接入了以下兼容端点：

- 视频：`/v1/videos`
- 图片生成：`/v1/images/generations`
- 图片编辑：`/v1/images/edits`

客户侧通常只需要把 `model` 改成新模型 ID。

上线新模型时建议平台维护方同步更新：

1. 模型广场模型 ID、计费方式、价格。
2. 模型能力矩阵：时长范围、分辨率、是否支持参考图/视频/音频/首尾帧。
3. 如果新模型不支持通用字段，需要在备注中明确说明。
4. 如有特殊参数，建议放到 `metadata` 或 `extra` 中，避免破坏通用请求结构。

## 11. 接入注意事项

- 参考素材必须是公网可访问 URL，不支持内网地址、本地路径或需要登录的链接。
- 视频和音频文件建议使用常见格式：`mp4`、`mov`、`webm`、`mp3`、`wav`、`m4a`。
- 图片建议使用：`jpg`、`jpeg`、`png`、`webp`。
- 批量任务请控制并发，避免触发限流或重复计费。
- 客户侧请保存 `task_id`、请求参数、提交时间、最终结果 URL，方便排查。
- 生产环境不要在浏览器前端明文暴露 API Key。
- 如果模型能力矩阵与模型广场显示不一致，以模型广场最新显示为准。
